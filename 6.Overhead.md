# 额外开销 （Overhead）

开销延迟（Overhead latency）是指在没有进行有效计算的情况下所花费的时间。

与受计算（compute）或内存（memory）瓶颈限制的时间不同，GPU 在受限下正全速工作，尽量完成计算或数据传输，而开销延迟表示 GPU 正在等待接收任务，没有执行有效工作。

开销通常来自 CPU 端瓶颈，导致 GPU 无法快速接收任务。

例如，CUDA API 调用开销每次内核启动约增加 10 微秒。此外，像 PyTorch 或 TensorFlow 这样的框架在决定启动哪个内核时也会消耗许多微秒。这里我们通常称之为“主机开销（host overhead）”，尽管这个术语并不完全标准化。CUDA Graphs 可以将多个设备端内核收集到一次主机端启动中，是常用的解决方案。更多内容可参见 GTC 2025 上的《CUDA Techniques to Maximize Concurrency and System Utilization》讲座。

“内存开销（Memory overhead）”或“通信开销（communications overhead）”是指在 CPU 与 GPU 或不同 GPU 之间传输数据时产生的延迟。但当通信带宽成为限制因素时，通常更好将其视作一种 内存受限（memory-bounded） 情况，只不过这里的“内存”分布在多台机器上。
